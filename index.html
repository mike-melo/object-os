What is ObjectOS and what is its goals?

  • An experiment
  • A learning experience on how to write a micro-kernel OS in x86 16-bit (for now) assembly from the ground up
  • An attempt at introducing object constructs into an OS as early as possible and to see how it works
  • Create a very basic object-oriented language that is essentially the CLI and interface for a programmer
  • Have a good time writing it

Some high level functionality being aimed for to get things going

  • A basic two-stage bootloader for the kernel (DONE)
  • A basic CLI (DONE)
  • A build process that integrates with VirtualBox to allow for easy regression and testing
  • Dynamic module loading and registry to allow interaction between modules (ie calling of procedures)
  • Breaking things down into modules that make sense for reuse
  • A language construct that allows for the creation of primitive string objects (DONE)
  • A very primitive dynamic heap (DONE)
  • Ability to associate labels or variable names with a created object
  • Storing captured input into an object
  • Some basic arithmetic operations where the result can be stored and referenced
  • A method like construct that can be attached to objects

The aim at this stage is to have a primitive CLI/language/API and some basic building blocks for a programmer to build on top of.

Ambitious? You betcha! But having a basic roadmap and some fun is a good start!
